Study Guide for Viva and Teacher Explanation

Use this page as your preparation sheet. It is written to help you explain the project confidently, step by step, with enough depth to answer follow-up questions.

1. One-Minute Introduction

“My project is called Game Score Evolution Simulator. It uses ordinary differential equations to model how two teams’ scores change over time. I compare several numerical methods such as Euler, Heun, Midpoint, Ralston, Adams-Bashforth 2, and RK4. The goal is to show how different solvers affect accuracy, runtime, and the realism of score prediction.”

2. Core Concept to Remember

Main mathematical idea

The model uses coupled ODEs because each team’s scoring rate affects the other team’s score evolution.

Why ODEs

ODEs are suitable because score changes continuously with time, and we want a trajectory instead of only a final result.

3. Full Formula Logic

1. drive = attack × momentum × tempo × saturation
2. pressure = 1 ± pressure_strength × lead_difference
3. clutch = periodic burst term based on time
4. defensive_drag = opponent_defense × coupling × current_score
5. final rate = max(0, drive × pressure + clutch - defensive_drag)

This formula structure is important because it explains that scoring is not random. It is influenced by offense, defense, current lead, and a saturation cap.

4. Method Comparison Notes

Method Say This
EulerIt is the simplest method and easiest to explain, but it is less accurate for nonlinear systems.
HeunIt corrects the Euler estimate by taking a second slope, so it improves stability.
MidpointIt estimates the slope at the midpoint, which usually gives better accuracy than Euler.
RalstonIt is a refined second-order method with better error behavior than standard RK2 choices.
Adams-Bashforth 2It uses previous derivative information, so it is efficient for repeated steps.
RK4It is the strongest accuracy method in this project and serves as the reference baseline.

5. Benchmark Terms You Must Explain Clearly

Trajectory RMSE It measures how closely the whole score path matches the reference path, not only the final score.
Runtime It shows how long the solver takes, which matters when a model must run quickly.
Efficiency score It combines accuracy and speed into one number so methods can be compared more easily.
Reference baseline A very fine-step RK4 run is used as the trusted comparison trajectory.

7. How to Answer Common Viva Questions

Why did you choose ODEs? Because ODEs are ideal for continuous change and let us model score growth over time.
Why compare many methods? To show the trade-off between accuracy, speed, and implementation complexity.
Why not only use RK4? Because numerical methods should be compared to show the improvement over simpler methods.
What makes this a game analytics project? It uses scoring, momentum, defense, and comeback measures to analyze match behavior.

8. Short Speaking Flow During Presentation

  1. Introduce the project title and objective.
  2. Explain the ODE model and why scores evolve continuously.
  3. Show the scenario presets and say that they represent different game styles.
  4. Demonstrate the method cards and mention the solver orders.
  5. Run the simulation and show the graph.
  6. Discuss which method is most accurate and which is fastest.
  7. Conclude with why the project is useful for numerical analysis and balancing studies.

9. Final Revision Reminder

Remember this core line: “The project compares numerical methods on a nonlinear score evolution ODE model to study how solver choice affects simulation realism, accuracy, and performance.”
